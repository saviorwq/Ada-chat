# 故障排查 / Troubleshooting

## 最短排查路径
1. 确认登录态有效
2. 检查供应商 Base URL / API Key
3. 获取模型并保存
4. 主界面确认分类/供应商/模型不为空
5. F12 查看 Network 返回码

## 常见错误
- `401/403`：未授权或登录失效
- `429`：限流，换模型或自动切换
- `404`：接口路径错误
- `500`：后端异常，查 PHP/Nginx 日志
- “AI 一轮后失忆”：确认当前版本为 `v1.1.0+`，并且没有第三方插件篡改请求历史
- “参数不生效”：检查当前模式是否为聊天模式，且供应商模型支持对应参数

## 服务器专项
- `ai_data/` 必须可写
- 必配 Nginx 敏感目录拦截

## 反馈时请附
- 分类 + 供应商 + 模型
- Base URL（可打码）
- 返回码和报错截图
